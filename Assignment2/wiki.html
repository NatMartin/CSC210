[edit] Common IPL Issues

  • see FAQ
  • Q: I follow the specs on sizes of the columns and the margins, but it still doesn't work.
    • A: When students put a 2% margin left and right, it should only be on the left. The margins in the middle do not overlap.
  • Q: There is a gap between my banner and the top and sides of the page. I have made it 100% wide and removed all padding and margins, why won't it go away?
    • A: The body tag has a default margin, so you must say body { margin: 0px; padding: 0px; } to remove that margin.
  • Q: How do I get the 33% rating to look right?
    • A: Tell them to play with any/all of: float, vertical-align (bottom), line-height.
  • Q: How do I get rid of the bullets on the ul? Can I change it to another tag?
    • A: No, don't change it. You can set list-style-type: none; on the ul. See the textbook Chapter 3 section about styles for lists.

[edit] Grading Checklist

(bold means not covered by Style Checker)

GENERAL GRADING GUIDELINE: They don't have to be pixel-perfect. If something is off by ~2-3 pixels or less, probably let it go unless it is really noticeable or ugly. If a given item has a major flaw that is not listed below, please mark off ALL points for the relevant category of points.

  • external correctness
    • borders, padding, margins (box model)
      • -1 if Jay Sherman's "it stinks" review is not tall enough (should have overflow: hidden; setting the div's height or min-height instead is a hack)
      • -1 too much spacing between each review
      • -1 body doesn't have 0 padding/margin
    • images and backgrounds
      • -1 top banner should have bannerbg (green background image) behind it
      • -1 missing favicon (have to pop out their page in its own window to see it)
    • otherwise correct appearance
      • incorrect fonts
      • -1 reviewers' publications should be italic; but reviewer's name should not
      • -1 large red text '33%' aligned improperly
      • -1 major issues in the "(1-10) of 88 reviews" banner at bottom (-1 if really minor, off by a few pixels)
  • internal correctness
    • tmnt.html - doesn't substantially modify skeleton other than div/span, class/id
      • -1 if they use <em> tags to italicize each review/quote (should be font-style:italic in CSS)
    • movie.css - no hacked layouts
      • -1 setting unnecessary properties (width/height 100%, random fonts/padding/margins/sizes that were not specified) (-1 per added random style value that we didn't specify, up to a max of -3)
    • movie.css - eliminates redundancy
      • -1 using lots of classes where a context selector would suffice
      • -1 left/right columns should share common styles (width, float, border, padding); shouldn't have a large rule for each column
      • -1 identical CSS rules for multiple things (such as reviewer images and name images, etc.)

[edit] Common Grading Inconsistencies

Issue 
Quick blurb of issue.
Symptoms 
The case(s) that is graded inconsistently.
Solution 
The grading policy we decided to follow for this case.
Inconsistency 
What is the discrepancy in grades.
Issue Symptoms Solution Inconsistency
Column spacing The horizontal spacing between columns or the vertical spacing between the columns and the rating banner is off by a few pixels (less than an 'm'). (OK) We decided to not take away any points for minor spacing issues having to do with the columns. Some TAs end up being too harsh and taking away the "two columns for reviews" point when there is only a few pixels difference.
Unnecessary div/span Students often make one or more of the following mistakes (but there are others)
  • Unnecessary <div> around their overview image
  • Unnecessary id="rating" in their <span> around the 33%
  • Unnecessary <div> for their footer, where just a <p> would've worked. Example: 
<div id="footer">
	 <p>(1-10) of 88</p>
</div>
(-1 "correct usage of tags, div/span, id/class") If the student does 2 or 3 of these things we should deduct a point. In other words they get one freebie extraneous thing. Some TAs deduct for one issue some do not deduct even if all three are present.
<em> for review publication The review publication is italicized, so some students try to achieve this by wrapping the review publication in an em tag. They should use a span and apply the style via css. (-1 "doesn't substantially modify skeleton other than div/span, class/id") Using em tags does lose this point. Some TAs simply miss this or do not consider it substantial enough.
Differences with webster file The students sometimes go back and edit their webster page without resubmitting to grade-it. (-1 uploaded correctly to webster) if the stuff changed is anything other than comments, regardless of modification date. Some TAs were a little more lenient and did not take off the point if there were only small differences.
